Як користуватись

  1. SMILES: встав рядок у поле SMILES та натисни Enter.
  2. Або використай Samples, щоб швидко завантажити готовий приклад.
  3. Файли: завантаж або перетягни .cif, .csv, .xyz, .json, .mol, .pdb/.ent або POSCAR/CONTCAR на сторінку.
  4. Для періодичних структур (CIF/POSCAR/JSON) за потреби введи Tiles (наприклад 2x2x1).
  5. Якщо в сцені кілька об’єктів, обери активний у списку Scene і редагуй лише його.

Формати вводу

Нижче описано, що саме очікується від кожного формату і які практичні нюанси важливі саме для TEM‑подібної візуалізації.

SMILES (рядок)

  • Підходить для молекул та іонних систем.
  • Топологія (зв’язки) задана у SMILES.
  • Координати генеруються через RDKit (WASM) у браузері.
  • Зручний варіант для швидких тестів, порівнянь і демонстрацій.

CIF (.cif / текст)

  • Підходить для кристалів і періодичних структур (unit cell + координати атомів).
  • Може містити дробові або декартові координати.
  • Часто не містить “хімічних” зв’язків — це нормально для кристалічних даних.
  • За потреби структура повторюється через Tiles (напр., 2x2x1, 3x3x2).

MOL / SDF (.mol / .sdf)

  • Підходить для малих молекул (органіка, фарм-ліганди), часто з баз даних.
  • Містить явні зв’язки: якщо вони задані — використовуються як є (guess не застосовується).
  • Координати можуть бути 2D або 3D; для 2D Z приймається 0 (для DoF це ок).
  • Найкращий формат, коли треба точно показати топологію зв’язків.

PDB (.pdb / .ent)

  • Підходить для біомакромолекул і комплексів (білки, ДНК/РНК, ліганди).
  • Координати читаються з ATOM/HETATM; можливі моделі/altLoc — щоб не було дублювання, береться основний варіант.
  • Зв’язки часто відсутні: якщо є CONECT — він використовується; інакше зв’язки можуть бути лише допоміжні.
  • Файли можуть бути великими: якщо повільно — зменш canvas або вимкни важкі ефекти.

POSCAR/CONTCAR (VASP)

  • Періодичні структури: містить вектори ґратки (cell) і координати атомів (Direct/Cartesian).
  • Підтримує Tiles: можна повторити комірку (2x2x1, 3x3x2) щоб побачити більший фрагмент кристалу.
  • Зв’язки зазвичай не задані — у візуалізаторі це допоміжний шар для читабельності.
  • Зручний формат для матеріалів/симуляцій; часто йде поруч із CIF.

XYZ (.xyz)

  • Простий список атомів у декартових координатах (зазвичай у Å); добре для молекул, кластерів, фрагментів.
  • Не містить зв’язків: за потреби “Зв’язки” працюють як приблизний допоміжний шар.
  • Не містить комірки: Tiles не застосовується.
  • Зручний як універсальний обмінний формат між програмами та симуляціями.

JSON (.json)

  • Обмінний формат: може містити atoms, bonds і/або cell (комірку).
  • Якщо є cell — це періодична структура і працює Tiles; якщо cell немає — це “молекулярний” режим.
  • Якщо bonds задані — вони використовуються як є; якщо ні — зв’язки можуть бути допоміжні.
  • Зручно для збереження/відтворення сцен і сумісності між різними джерелами даних.

CSV (.csv)

  • Табличний список атомів (наприклад колонки element/sym + x,y,z).
  • Потрібні принаймні елемент і координати; зайві колонки ігноруються.
  • Зв’язки/комірка не задаються: Tiles не застосовується; “Зв’язки” — допоміжна візуалізація.
  • Корисно для власних даних з Excel/Google Sheets або зі скриптів.

Нотатка: у структурних форматах (особливо CIF) зв’язки можуть бути відсутні або не збігатися з “молекульним” уявленням. У цьому симуляторі зв’язки — допоміжний шар візуалізації, а не “істина” хімії.

Пояснення елементів керування

Регулятори змінюють вигляд TEM‑подібного зображення (контраст, фон, шум, розфокус), але не “переписують” саму структуру.

Переміщення/обертання

  • ↑ ↓ ← → — панорамування (зсув) в’юхи по площині кадру.
  • ⟲ / ⟳ — обертання “столика” (в’юхи) навколо центру сцени.
  • Reset — скидання позиції/обертання у базовий стан.
  • ⨀ у Move скидає позицію об’єкта, ⨀ у Rotate скидає його обертання.
  • Кнопки Z− / Z+ рухають активний об’єкт уздовж осі Z сцени.

Перемикачі

Інверсія (cb-invert)
Інвертує яскравості: темне стає світлим і навпаки.
Шум (cb-noise)
Вмикає додавання шуму детектора. Сила задається слайдером “Шум (σ)”.
Зв’язки (cb-bonds)
Показує/ховає зв’язки між атомами. У CIF зв’язків може не бути.
Ховати передні шари при розфокусі (cb-hidefront)
Допоміжний режим для DoF: може ховати те, що “перед” фокусом, щоб шари менше перекривалися.
Масштаб (cb-scale)
Показує/ховає масштабну лінійку (якщо вона ввімкнена у рендері).

Слайдери

Å/pix (rng-zoom)
Зум у фізичних одиницях. Менше — наблизити, більше — віддалити.
Контраст (rng-contrast)
Підсилює різницю між темними й світлими ділянками.
Розмиття (σ) (rng-blur)
Загальне після-розмиття фінального зображення.
Фон (0–255) (rng-bg)
Сірість фону: 0 — чорний, 127 — нейтральний, 255 — білий.
Шум (σ) (rng-noise)
Сила шуму. Має сенс, якщо ввімкнений перемикач “Шум”.
Фокус Z (норм.) (rng-focus)
Положення фокусу по глибині (нормалізовано). Використовується разом із DoF.
Розфокус (Å⁻¹) (rng-dof)
Сила різного розмиття шарів по глибині.
Ширина дуги (px) (rng-bwidth)
Візуальна ширина “перешийків/дуг” зв’язків.
Висота дуги (rng-bamp)
Амплітуда “хвилі” зв’язку (стилізація зв’язків).

Поля вводу та кнопки

Підлога/Стеля яскравості (tb-clip-lo / tb-clip-hi)
Обрізання діапазону яскравості після рендера.
Tiles (tb-tiles)
Повторення комірки для періодичних структур: 2x2x1, 3x3x2 тощо.
SMILES (tb-smiles)
Рядок SMILES. Enter запускає перебудову та рендер.
Завантажити файл (file-input)
Локальне завантаження/перетягування файлів (.cif, .poscar, .json, .xyz, .mol, .pdb/.ent, .csv). Файл не відправляється на сервер.
Ширина/Висота (tb-w / tb-h)
Розмір canvas. Впливає на детальність та швидкість.
Експортувати PNG (btn-export)
Зберігає поточне зображення в PNG.
lbl-title
Назва/ідентифікація поточного об’єкта (якщо є в даних).

Сцена та робота з кількома об’єктами

  • Кожен завантажений файл або SMILES додається як окремий об’єкт сцени.
  • У панелі Scene можна перемикати активний об’єкт, вмикати/вимикати його видимість і видаляти непотрібні елементи.
  • Лічильник Scene показує, скільки об’єктів зараз реально беруть участь у рендері.
  • Кнопка Reset edits: object скидає лише активний об’єкт, а Reset edits: all — усі об’єкти сцени.

Редагування активного об’єкта

  • Блок Move змінює положення активного об’єкта по X, Y та Z у глобальній сцені.
  • Блок Rotate обертає активний об’єкт навколо осей X, Y і Z незалежно від інших об’єктів.
  • Під кнопками показуються поточні числові значення зсуву та кутів обертання.
  • У Additional object edit доступний Z-clip: можна обрізати активний об’єкт за глобальною віссю Z після повороту та зсуву.
  • Z-clip має незалежні межі min/max і окрему кнопку Reset clip.

Режими TEM / STM / AFM

  • TEM — проєкційний режим: внесок атомів накопичується по товщині зразка.
  • STM — поверхневий режим: працює як верхньошаровий сигнал, без DoF і без зв’язків.
  • AFM — також поверхневий режим; замість TEM-логіки тут важливі рельєф і параметр Tip sharpness.
  • Scanlines з’являються тільки у STM, а Tip sharpness — тільки в AFM.

Експорт і запис

  • Export PNG зберігає поточний кадр canvas у вигляді растрового зображення.
  • Export XYZ вивантажує один XYZ-файл для всіх видимих об’єктів після застосування їхніх pos/rot/Z-clip.
  • GIF tools дозволяють почати запис, зупинити його та зберегти готовий GIF-файл.
  • Поля Width/Height змінюють розмір canvas і впливають як на швидкість, так і на рівень деталізації.

Додаткові налаштування

  • Additional settings містить Background, параметри зв’язків і clipping яскравості floor/ceiling.
  • Element overrides дозволяє окремо задавати Size× і Darkness× для кожного елемента.
  • У таблиці overrides є прев’ю атома з поточними глобальними ефектами.
  • Export config / Import config зберігає і відновлює JSON-конфіг overrides.

Рекомендований сценарій роботи

  1. Швидко перевірити рендер на SMILES або Samples (контраст/фон/шум).
  2. Для кристалів — відкрити CIF/POSCAR/JSON, виставити Tiles і підібрати Å/pix.
  3. Якщо об’єктів кілька — спочатку відрегулювати їхні pos/rot/clip у Scene, а вже потім робити фінальний рендер.
  4. Для обміну геометрією — експортувати XYZ, для зображень — PNG, для анімації — GIF.

Діагностика

  • Порожнє зображення: перевір SMILES/файл, зменш Å/pix (наблизь), підніми контраст.
  • Занадто темно/світло: підкоригуй фон і контраст, за потреби clip lo/hi.
  • Дивні зв’язки у CIF: у CIF зв’язків часто немає; зв’язки тут можуть бути допоміжні.
  • Підвисання: зменш canvas, зменш tiles, вимкни DoF/зв’язки/шум або “віддалися” (збільш Å/pix).
  • Якщо об’єкт “зник” у сцені — перевір його visibility, позицію по Z та межі Z-clip.

Бібліотеки файлів (де брати структури)

Матеріали та кристали (CIF / POSCAR)

  • Materials Project — велика база матеріалів (експорт структур, розрахункові дані).
  • Materials Cloud — архів структур і розрахунків, багато даних для завантаження.
  • Crystallography Open Database (COD) — відкрита база CIF-структур.
  • AFLOWlib — база матеріалів та інструменти, часто доступні структурні файли.
  • OQMD — база матеріалів (для багатьох записів доступний POSCAR).

Малі молекули (SMILES / MOL / SDF / XYZ)

  • PubChem — SMILES, SDF та довідкова інформація.
  • ChEMBL — біоактивні сполуки, експорти структур.
  • ZINC / CartBlanche — великі набори молекул (SMILES/SDF тощо).
  • NCI/CADD Chemical Identifier Resolver (CACTUS) — може віддати структуру у XYZ (та інших форматах) за назвою/SMILES/InChIKey; зручно для швидких XYZ‑файлів.
  • ChemSpider — пошук і довідкова база сполук; зручно знаходити структури/ідентифікатори і переходити до джерел.
  • WebQC Molecular Formats Converter — онлайн‑конвертер форматів (SMILES/MOL/SDF/XYZ та інші); швидко перетворює структуру у потрібний файл.

Біоструктури (PDB / mmCIF)

  • RCSB Protein Data Bank (PDB) — PDB/mmCIF для білків, ДНК/РНК, комплексів.

Довідка та референси

Бібліотеки

  • RDKit — бібліотека хемінформатики (у проєкті використовується WASM-збірка).

Специфікації форматів

  • IUCr CIF — офіційна інформація про формат CIF.
  • wwPDB file formats — документація PDB/mmCIF.

Референс-візуалізатори

  • 3Dmol.js
  • JSmol/Jmol

Нотатка: проєкт імітує TEM‑подібний вигляд для навчальної/демонстраційної візуалізації і не є повноцінним multislice‑моделюванням.
